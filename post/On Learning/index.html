Concepts

…and what matters most is Understanding. And Understanding starts with “What Matters” - Quora: Forays into AI because we get better with practice as we get more experience, which is a hallmark of any Intuition based skill - Reductionism Considered Harmful This is another hallmark of Intuition based skills: they cannot be taught as high level rules, they have to be experienced bottom up. This is, incidentally, also the difference between Teaching and Coaching - [ibid]

  • Using features of the information to classifying & divide concepts.

  • Begin with the problem space. Overview of “Why are we learning this? What are the fundamental problems this is trying to address?” Try and use a concrete example; ground it.

    • Why do we use databases? We need ways to manage incredibly large sets of data and its metadata. Metadata can include the relationships within data.
      • This grounds the upcoming concept against the student’s pre-existing patterns.
    • What problems do databases have to solve? Writing to disk is very slow compared to the rest of I/O, so all databases need to optimize this. How they do so is a significant difference between databases.
      • We can use this difference as a means of classifying and comparing different databases.

  • Provide a framework for comparison

    • Would listing up-front the variables for comparison overwhelm or assist? Perhaps this is size-dependent on the number of variables.

  • Use a common example across the different categories

    • For databases, have a given set of data, and depict how it’s stored in different DBs.
    • In military terms, this would be the “walk” phase - hand-holding still involved. The “run” might be creating a data set, and having the student depict the different style of storage.

Techniques

Tim Ferris promotes DSSS and it’s sub-steps, CaFE

  • DSSS
    1. Deconstruction
      • Reducing
      • Interviewing
      • Reversal
      • Translating
    2. Selection (The 80⁄20 rule)
    3. Sequencing (The order in which you choose to learn)
    4. Stakes (The carrot & the stick)
  • CaFE
    1. Compression
      • How can I represent the essential 20% in a compact reference?
    2. Frequency
      • What is most sustainable schedule I can manage for learning this information?
    3. Encoding
      • How can I leverage anchors, connections, and whatever tricks possible to make this information stick?
